## Проекты

### Continuum Memory — архитектура непрерывной памяти для LLM и диалоговых систем

**Описание:**  
Концепция и архитектура долговременной памяти нового поколения для AI-систем, предназначенная для сохранения релевантного контекста на длинной дистанции общения без деградации качества ответов. Проект ориентирован на AI-ассистентов, игровые NPC, диалоговые движки и корпоративные LLM-продукты, где критична способность системы удерживать факты, связи, пользовательские предпочтения и историю взаимодействия на протяжении сотен и тысяч сообщений.

**Роль и достижения:**
- Спроектировал архитектуру **непрерывной памяти**, в которой контекст проходит через этапы отбора, компрессии, приоритизации и долговременного закрепления вместо механического обрезания по длине окна.
- Разработал подход к сохранению **семантически значимых фактов и событий** в сверхдлинных диалогах, минимизируя потерю контекста, характерную для стандартных LLM-архитектур.
- Продумал memory-слой как самостоятельный интеллектуальный механизм, который:
  - выделяет важные сущности, факты и сюжетные узлы;
  - определяет их приоритет;
  - распределяет информацию между краткосрочным, долгосрочным и компрессированным хранением.
- Сфокусировался на повышении **устойчивости качества ответов** на длинной дистанции, чтобы система сохраняла критические детали даже после десятков и сотен сообщений.
- Спроектировал Continuum Memory как **API-first решение**, пригодное для встраивания в существующие чат-боты, игровые движки и AI-платформы в качестве отдельного memory-слоя.
- Проработал проект не только как исследовательскую концепцию, но и как **инфраструктурный продукт**, в котором долговременная память выступает прямым преимуществом по качеству, консистентности и удержанию пользователей.
- Сформировал бизнесовую и инженерную рамку идеи: целевая аудитория — разработчики AI-ассистентов, игровые студии и IT-компании, работающие с LLM; формат внедрения — API и инфраструктурный компонент.

**Технологическая ценность проекта:**
- Решает одну из ключевых проблем современных диалоговых моделей — быструю потерю контекста.
- Повышает консистентность, связанность и персонализированность ответов.
- Открывает возможность создавать AI-системы с ощущением непрерывности памяти о пользователе, мире и истории взаимодействия.
- Особенно ценен для продуктов с длинными сессиями: игровых сценариев, AI-компаньонов, корпоративных помощников, обучающих систем и интеллектуальных интерфейсов.

### HandTalk — платформа перевода на жестовый язык

**Описание:**  
Платформа для доступной коммуникации, объединяющая перевод звук/текст → жестовый язык с визуализацией через 3D-аватар, а также двусторонний сценарий перевода для полноценного диалога.

**Роль и вклад:**
- Разработал backend-платформу, включая серверную логику, API и интеграцию клиентской части с сервисами перевода.
- Участвовал в обучении ML-модели для перевода жестов в текст: подготовка данных, построение пайплайна обучения и тестирование качества.
- Поддерживал модульную архитектуру, позволяющую независимо развивать отдельные алгоритмы и компоненты системы.
- Обеспечивал связность backend- и ML-части, формируя целостное прикладное решение.

### DND_AI — интеллектуальная игровая экосистема

**Описание:**  
Система, объединяющая нарратив Dungeons & Dragons с диалоговым ИИ, способным поддерживать сверхдолгие последовательные сессии без потери контекста, личности и сюжетной целостности.

**Роль и достижения:**
- Разработал каскадную систему памяти с многоуровневым стеком: краткосрочный, среднесрочный, долгосрочный и «бесконечный» слои.
- Использовал различные подходы к компрессии и представлению информации, от extractive summarization до hybrid abstractive методов.
- Спроектировал механики приоритизации, иммунитета к удалению и циклической очистки для поддержки релевантной памяти при тысячах сообщений.
- Реализовал концепцию **Infinity Memory** как когнитивный слой для хранения фактов, событий и связей с алгоритмами долгосрочного удержания ключевой информации.
- Интегрировал модульную API-ориентированную архитектуру взаимодействия между диалоговым движком, памятью, поиском и модулем обобщения.
- Прорабатывал задачу сохранения личности ИИ и сюжетной непрерывности в длинных игровых сессиях.

**Роль и достижения:**

- Разработал backend-часть: вычислительную логику, API/эндпоинты и интеграцию с веб-интерфейсом.
- Обеспечил воспроизводимость результатов и корректную обработку входных данных.
